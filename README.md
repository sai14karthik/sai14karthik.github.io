## Deployment to GitHub Pages

### Option 1: Using GitHub Repository (Recommended)

1. **Create a new repository** on GitHub (e.g., `portfolio` or `sai14karthik.github.io`)

2. **Clone the repository** to your local machine:
   ```bash
   git clone https://github.com/yourusername/portfolio.git
   cd portfolio
   ```

3. **Copy all files** from this directory to the repository:
   - `index.html`
   - `stylesheet.css`
   - `README.md`

4. **Commit and push**:
   ```bash
   git add .
   git commit -m "Initial portfolio website"
   git push origin main
   ```

5. **Enable GitHub Pages**:
   - Go to your repository on GitHub
   - Click on **Settings** → **Pages**
   - Under **Source**, select **Deploy from a branch**
   - Choose **main** branch and **/ (root)** folder
   - Click **Save**

6. **Your site will be live at**:
   - `https://yourusername.github.io/portfolio/` (if repo is named `portfolio`)
   - OR `https://yourusername.github.io/` (if repo is named `yourusername.github.io`)

### Option 2: Using GitHub Desktop

1. Open GitHub Desktop
2. Create a new repository
3. Copy the files to the repository folder
4. Commit and push
5. Enable GitHub Pages in repository settings

## Customization

### Update Personal Information

Edit `index.html` to update:
- Contact information
- Experience details
- Education
- Projects
- Publications
- Skills
- Achievements

### Update Styling

Modify `stylesheet.css` to customize:
- Colors
- Fonts
- Spacing
- Layout

### Update SEO Information

In `index.html`, update the meta tags in the `<head>` section:
- `meta name="description"`
- `meta name="keywords"`
- Open Graph tags
- Structured data (JSON-LD)

## SEO Optimization

This portfolio includes:

1. **Meta Tags**: Description, keywords, author
2. **Open Graph Tags**: For social media sharing
3. **Structured Data**: JSON-LD schema for Google
4. **Semantic HTML**: Proper heading hierarchy
5. **Canonical URL**: Prevents duplicate content issues
6. **Mobile Responsive**: Important for Google rankings
